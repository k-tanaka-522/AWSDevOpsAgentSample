# 要件定義書 (Requirements Definition)

## 1. プロジェクト概要
### 1.1 プロジェクト名
Task Manager Pro (T-Pro)

### 1.2 背景と目的
現代のナレッジワーカーは多数のタスクに追われており、「どれから着手すべきか」の判断に時間を要している。
本プロジェクトでは、従来の期日管理に加え、「影響度」と「工数」を考慮した高度な優先度決定ロジック（Priority Score）を導入し、
ユーザーが「今、最もやるべきこと」を即座に把握できる次世代タスク管理アプリケーションを開発する。

### 1.3 ターゲットユーザー
- 複数のタスクを並行して抱えるビジネスパーソン
- タスクの優先順位付けに課題を感じているエンジニア/マネージャー

## 2. 機能要件
### 2.1 必須機能 (Must-Have)
| ID | 機能区分 | 機能名 | 詳細 |
|---|---|---|---|
| REQ-001 | タスク管理 | タスク作成 | タイトル、説明、期日、工数、カテゴリーを設定してタスクを作成できること |
| REQ-002 | タスク管理 | タスク編集 | 既存タスクの属性を変更できること |
| REQ-003 | タスク管理 | タスク削除 | 不要なタスクを削除できること（論理削除推奨） |
| REQ-004 | タスク管理 | タスク完了 | タスクを完了状態にできること |
| REQ-005 | 優先度計算 | スコア算出 | 独自の計算式に基づき、タスク追加・更新時にリアルタイムで優先度スコアを計算すること |
| REQ-006 | 分類 | カテゴリ管理 | タスクをプロジェクトやコンテキストごとに分類できること |
| REQ-007 | 表示 | リスト表示 | 計算された優先度スコア順でタスクをソート表示できること |
| REQ-008 | 永続化 | データ保存 | アプリ終了後もデータが保持されること（SQLite使用） |

### 2.2 推奨機能 (Should-Have)
| ID | 機能区分 | 機能名 | 詳細 |
|---|---|---|---|
| REQ-009 | UI/UX | ダークモード | OS設定またはアプリ内設定でテーマ切り替えが可能であること |
| REQ-010 | 分析 | 統計表示 | 完了タスク数や進捗状況を可視化すること |

## 3. 非機能要件
| 項目 | 要件 |
|---|---|
| 性能 | タスク数1000件程度でもリスト表示が1秒以内に完了すること |
| 信頼性 | データベースファイルの破損対策（ジャーナルモード等） |
| 操作性 | モダンなWPF UI (Fluent Design準拠) 、レスポンシブなウィンドウ操作 |
| 保守性 | MVVMパターンを採用し、ViewとLogicが分離されていること |
| 拡張性 | 将来的なクラウド同期を見据えたアーキテクチャであること |

## 4. 環境要件
- **OS**: Windows 10/11 (64bit)
- **Framework**: .NET 9.0 (Preview/RC含む)
- **DB**: SQLite
