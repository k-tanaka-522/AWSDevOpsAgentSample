# R-01 業務要件定義書

## 1. はじめに
本ドキュメントは、「Task Manager Pro (T-Pro)」における業務要件を定義するものである。
現状のタスク管理業務の課題を整理し、本システム導入後の業務フロー（To-Be）を明確化する。

## 2. 業務概要
### 2.1 業務の目的
- 個人のタスク管理業務において、「優先度の判断」にかかる時間を削減し、最も価値のある活動にリソースを集中させる。
- 期限切れやタスクの抜け漏れを防止する。

### 2.2 対象業務範囲
- タスクの発生（起票）から完了（クローズ）までのライフサイクル管理。
- 日次/週次のタスク棚卸し業務。

## 3. 現状業務フロー (As-Is)
現状は、テキストファイル、付箋、あるいは脳内メモリに依存して管理しているケースを想定する。

1. **タスク発生**: 依頼や思いつきでタスクが発生。
2. **記録**: ノートやテキストにメモ。期日などは書かれたり書かれなかったりする。
3. **着手判断**: リストを上から眺め、気になったものや期限が迫っているものを着手。**重要だが緊急でないタスク**が後回しにされがち。
4. **完了**: 線で消す。

**課題**:
- どれから手を付けるべきか迷う時間がムダ。
- 「緊急度」と「重要度」の区別がつかず、締め切り駆動になりがち。

## 4. 新業務フロー (To-Be)
本システム導入後の業務フロー。

### 4.1 業務フロー図
```mermaid
graph TD
    Start((開始)) --> Input[タスク登録]
    Input --> Think[評価・入力<br/>(工数/影響度/緊急度)]
    Think --> SystemCalc[[システムによる<br/>優先度自動計算]]
    SystemCalc --> CheckList[リスト確認]
    CheckList --> Select[Topタスク着手]
    Select --> Work[作業実施]
    Work --> Comp{完了?}
    Comp -- No --> Update[進捗/ステータス更新]
    Comp -- Yes --> Done[完了としてマーク]
    Update --> SystemCalc
    Done --> End((終了))
```

### 4.2 業務シナリオ
#### シナリオ1: 朝のタスク確認
1. ユーザーは T-Pro を起動する。
2. 「今日やる」フィルターを適用する。
3. 優先度スコア(P)が高い順にソートされたリストを確認する。
4. スコアTop 3 のタスクをその日の注力タスクとして認識し、着手する。

#### シナリオ2: 突発タスクの登録
1. 上司から急な依頼が入る。
2. 「クイック追加」機能でタスクを登録する。
3. 暫定的に緊急度(Urgency)を「高」に設定する。
4. システムが即座に高スコアを算出し、リスト上部に割り込ませる。
5. ユーザーは現在の作業を中断すべきか、スコアを見て判断する。

## 5. 用語定義
| 用語 | 定義 |
|---|---|
| **影響度 (Impact)** | そのタスクを達成した際に得られる成果の大きさ、あるいはビジネスへの貢献度。1-10で数値化。 |
| **緊急度 (Urgency)** | 時間的な切迫度。期日とは別に、対応の急ぎ具合を示す。1-10で数値化。 |
| **工数 (Effort)** | タスク完了にかかる推定時間（時間単位）。分母となるため、小さいほど生産性（コスパ）が高いとみなされる。 |
| **優先度スコア (Priority Score)** | システムが算出する着手推奨度。数値が高いほど優先すべき。 |
