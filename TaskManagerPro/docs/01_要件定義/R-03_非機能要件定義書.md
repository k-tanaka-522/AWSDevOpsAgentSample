# R-03 非機能要件定義書

## 1. はじめに
本ドキュメントは、「Task Manager Pro」における非機能要件（品質要求）を定義する。
IPA「非機能要求グレード」の「中小規模システム」モデルをベースに調整を行う。

## 2. 非機能要件一覧

### 2.1 可用性 (Availability)
| 項目 | 要件 | 備考 |
|---|---|---|
| 稼働時間 | 24時間365日（デスクトップアプリのためユーザーのPC稼働に依存） | スタンドアロンアプリ |
| 障害回復 | アプリケーションクラッシュ時、再起動により直前の保存状態（SQLiteコミット済みデータ）から再開できること。 | |
| データ保全 | SQLiteのジャーナルモード(WAL)を使用し、電源断などによるファイル破損リスクを低減する。 | |

### 2.2 性能・拡張性 (Performance & Scalability)
| 項目 | 要件 | 備考 |
|---|---|---|
| 応答時間 | リスト表示、タスク保存などの主要操作は **0.5秒以内** に応答すること。 | UIスレッドをブロックしない非同期処理 |
| スループット | 特になし（シングルユーザー）。 | |
| データ量 | タスク総数 **10,000件** まではパフォーマンス低下なく動作すること。 | インデックス設計での担保 |

### 2.3 運用・保守性 (Operation & Maintenance)
| 項目 | 要件 | 備考 |
|---|---|---|
| ログ出力 | 例外発生時、アプリケーションディレクトリ内の `logs/app.log` にスタックトレースを出力すること。 | 調査用 |
| インストール | XCopy配布、またはインストーラーによる簡易セットアップが可能であること。 | .NET Runtime依存の整理 |

### 2.4 セキュリティ (Security)
| 項目 | 要件 | 備考 |
|---|---|---|
| アクセス制御 | 特になし（ローカルPCのOSログオン認証に依存）。 | |
| データ保護 | DBファイルへのアクセス権限はOSのファイルシステムの権限管理に従う。 | |

### 2.5 ユーザーインターフェース (UI/UX)
| 項目 | 要件 | 備考 |
|---|---|---|
| デザイン | Microsoft Fluent Design System に準拠し、モダンで直感的な操作感を提供すること。 | |
| テーマ | ダークモードに対応し、OS設定またはアプリ設定で切り替え可能であること。 | 長時間作業への配慮 |
| レスポンシブ | ウィンドウサイズ変更時にレイアウトが崩れず、適切に追従すること。 | |

## 3. 技術制約
- **フレームワーク**: .NET 9.0
- **UIライブラリ**: WPF (Windows Presentation Foundation)
- **データベース**: SQLite 3
- **ORM**: Entity Framework Core 9.0
